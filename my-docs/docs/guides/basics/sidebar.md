# Configuring the Sidebar

The sidebar provides hierarchical navigation for your documentation. Learn how to configure and customize it.

## Sidebar File

Create a `sidebars.js` file in your project root to define sidebar structure.

```javascript
module.exports = {
  tutorialSidebar: [
    'intro',
    'getting-started/installation',
    'getting-started/configuration',
  ],
};
```

## Sidebar Items

### Document Items

Link to individual documentation pages:

```javascript
'intro'
```

This corresponds to `docs/intro.md`.

### Category Items

Group related documents:

```javascript
{
  type: 'category',
  label: 'Getting Started',
  items: [
    'getting-started/installation',
    'getting-started/configuration',
    'getting-started/first-project',
  ],
}
```

Categories can be nested and collapsed.

### Link Items

Add external links:

```javascript
{
  type: 'link',
  label: 'GitHub',
  href: 'https://github.com/my-org/my-repo',
}
```

### Auto-generated Sidebars

Automatically generate sidebar from file structure:

```javascript
tutorialSidebar: [
  {
    type: 'autogenerated',
    dirName: '.',
  },
]
```

This creates categories based on folder structure.

## Multiple Sidebars

You can have multiple sidebars for different sections:

```javascript
module.exports = {
  tutorialSidebar: [
    'intro',
    'tutorial-basics/create-a-document',
  ],
  apiSidebar: [
    'api/cli',
    'api/config',
  ],
};
```

## Sidebar Configuration Options

### Category Options

```javascript
{
  type: 'category',
  label: 'Advanced',
  collapsible: true,
  collapsed: false,
  items: [
    'advanced/deployment',
    'advanced/versioning',
  ],
}
```

- `collapsible`: Whether the category can be collapsed (default: true)
- `collapsed`: Whether the category starts collapsed (default: true for auto-generated)

### Custom Labels

Override the default label:

```javascript
{
  type: 'doc',
  id: 'getting-started/installation',
  label: 'Install Docusaurus',
}
```

## Sidebar Styling

Customize sidebar appearance with CSS:

```css
/* Custom sidebar styles */
.theme-doc-sidebar-container {
  /* Your styles */
}
```

## Sidebar Behavior

### Active Item Highlighting

Docusaurus automatically highlights the current page in the sidebar.

### Mobile Sidebar

On mobile devices, the sidebar becomes a slide-out menu.

### Search Integration

The sidebar works with search functionality to highlight relevant sections.

## Advanced Sidebar Features

### Dynamic Sidebars

Create sidebars programmatically:

```javascript
const fs = require('fs');

module.exports = {
  tutorialSidebar: fs.readdirSync('./docs/tutorial')
    .filter(file => file.endsWith('.md'))
    .map(file => `tutorial/${file.replace('.md', '')}`),
};
```

### Conditional Items

Show items based on environment:

```javascript
module.exports = {
  tutorialSidebar: [
    'intro',
    process.env.NODE_ENV === 'development' && 'debugging',
    'deployment',
  ].filter(Boolean),
};
```

## Best Practices

- Keep sidebar depth to 2-3 levels
- Use descriptive category labels
- Order items logically
- Test sidebar on different screen sizes
- Use auto-generated sidebars for large documentation sets
- Keep sidebar items concise

## Troubleshooting

### Sidebar Not Showing
- Check that `sidebars.js` exists and is properly formatted
- Verify sidebar ID matches the one in navbar configuration
- Ensure document files exist

### Wrong Order
- Items appear in the order defined in `sidebars.js`
- Use arrays to control order

### Links Not Working
- Check that document IDs match file paths
- Ensure files are in the correct directory

For more information, see the [Sidebar documentation](https://docusaurus.io/docs/sidebar).
